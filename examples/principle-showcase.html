Summary

  • Purpose: Keep generated HTML stable, reusable, and easy to validate.

Context

The renderer should produce consistent documents without copying large templates into model context.

Principles Or Features

Consistent output

Use one shared shell and semantic classes.

On-demand composition

Render only selected components.

Scenario skeletons

Use configured section order.

Reuse first

Prefer existing components and schema fragments.

Examples

python3 scripts/render_html.py --input doc.json --output doc.html --validate
