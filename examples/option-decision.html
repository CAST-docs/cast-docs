Renderer implementation decision

Updated 2026-05-13

Context

  • Decision: Choose how the HTML renderer should transform document JSON.

Decision Criteria

  • Deterministic output
  • Low token usage
  • Simple validation
  • Easy component reuse

Options Considered

The main options are direct HTML assembly and typed block rendering.

Comparison

OptionStrengthRisk
Direct HTML assemblyFast to prototypeHarder to validate
Typed block renderingPredictable and reusableRequires upfront schema

Decision

Decision

Use typed block rendering backed by configuration registries.

Tradeoffs

The schema costs more upfront but reduces drift once more scenarios are added.

Follow-ups

  1. Every configured scenario has a fixture.
  2. Renderer rejects unknown block types.

Implement schema validation

Wire doc.schema.json into validate_doc_json.py.
