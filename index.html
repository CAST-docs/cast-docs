CAST Docs

Owner CAST-docs · Updated 2026-05-26

Overview

  • Format: Author engineering documents as structured JSON; render to one self-contained HTML file.
  • Output: Inline CSS, no CDN, no external scripts. Opens in any browser, attaches to email, archives, and publishes to GitHub Pages.
  • Use as: A Claude Code skill, or a standalone Python renderer with zero third-party dependencies.

CAST Docs is a controlled replacement for Markdown-based engineering documents. The JSON is the source. The HTML is the artifact. The renderer enforces one visual system, one class vocabulary, and one validation contract across every output.

Five built-in scenarios

Each scenario picks a section order and a set of components that fit a common writing situation. Open any preview to see a rendered example; the matching JSON fixture lives next to it in the repository.

  • Problem investigation — Tracking down a bug, regression, or anomaly.
  • Option decision — Comparing options and recording why one was chosen.
  • Cross-team alignment — Aligning multiple teams on scope and ownership.
  • Document digest — Summarizing a long document or set of documents.
  • Principle showcase — Curating principles or guidelines for reuse.

Why this shape

HTML, not Markdown

Markdown cannot carry callouts, decision matrices, diff blocks, diagram viewers, and controlled tables without per-renderer extensions. Owning the HTML produces consistent output across every viewer.

JSON intermediate

Prose to HTML by a model is non-deterministic. Prose to JSON to HTML separates content from rendering, keeps the JSON validatable and diffable, and lets the renderer produce byte-stable output.

Self-contained output

Inline CSS, no external assets. The HTML survives being opened in any browser, attached to email, copied to S3, or served from GitHub Pages.

Config-driven renderer

Adding a new component, theme, layout, or interaction is config plus an asset file, not a Python branch. Visual and structural rules live in config/. The Python in scripts/ only assembles.

Get started

Requires Python 3.9 or newer. Standard library only. Clone the repository and render one of the bundled examples:

git clone https://github.com/CAST-docs/cast-docs.git
cd cast-docs
python3 scripts/render_html.py \
  --input examples/problem-investigation.json \
  --output out.html \
  --validate
Use as a Claude Code skill

Clone into ~/.claude/skills/cast-docs and the skill activates when you ask Claude for an engineering spec, decision record, or self-contained HTML document. References in references/ are loaded on demand.

  • README — Installation, validation pipeline, repository layout.
  • JSON schema — The contract every input document must satisfy.
  • Design references — Design laws, generation contract, module architecture, interactive features.
