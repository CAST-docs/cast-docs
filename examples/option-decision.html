Renderer implementation decision渲染器实现决策

Context背景

  • Decision决策: Choose how the HTML renderer should transform document JSON.选择 HTML 渲染器应如何转换文档 JSON。

Decision Criteria决策标准

  • Deterministic output确定性输出
  • Low token usage低 token 使用量
  • Simple validation简单校验
  • Easy component reuse组件易复用

Options Considered备选方案

The main options are direct HTML assembly and typed block rendering.主要方案是直接组装 HTML，或使用类型化 block 渲染。

Comparison对比

Option方案Strength优势Risk风险
Direct HTML assembly直接组装 HTMLFast to prototype原型实现快Harder to validate更难校验
Typed block rendering类型化 block 渲染Predictable and reusable可预测且可复用Requires upfront schema需要先定义 schema

Decision决策

Decision决策

Use typed block rendering backed by configuration registries, not direct HTML assembly.采用由 配置注册表 支撑的类型化 block 渲染，而不是 直接组装 HTML.

Tradeoffs取舍

The schema costs more upfront but reduces drift once more scenarios are added.schema 前期成本更高，但在添加更多场景后可以减少漂移。

Follow-ups后续事项

  1. Every configured scenario has a fixture.每个已配置场景都有 fixture。
  2. Renderer rejects unknown block types.渲染器会拒绝未知 block 类型。

Implement schema validation实现 schema 校验

Wire doc.schema.json into validate_doc_json.py.将 doc.schema.json 接入 validate_doc_json.py.
