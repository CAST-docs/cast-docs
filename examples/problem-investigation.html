Checkout latency spike investigation

Updated 2026-05-13

Background

  • Scope: Checkout requests showed intermittent latency spikes.
  • Window: Observed during the latest traffic peak.

This document records the investigation path and follow-up actions.

Problem Symptoms

Symptom

P95 latency increased while success rate stayed mostly stable.

Impact Scope

AreaImpactConfidence
Checkout APIHigher latencyhigh
Payment callbackNo direct impactmedium

Investigation Path

Core Logs And Evidence

Representative log
checkout latency=1840ms dependency=inventory status=200
  • services/checkout/inventory_client.go:88 — Dependency call without timeout override

Root Cause

Root cause

The checkout path inherited a longer dependency timeout than intended.

Mitigation And Fix

Set checkout-specific timeout

Use a bounded timeout for inventory reservation.

Inspect checkout inventory calls and propose a timeout change with tests.

Validation

  1. P95 latency returns to the normal band.
  2. Timeout behavior is covered by unit tests.

Prevention Measures

  • Add dependency timeout review to service checklist.
  • Alert on dependency latency before checkout P95 regresses.

Open Questions
