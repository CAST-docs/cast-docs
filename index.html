Overview概览

  • Format格式: Author engineering documents as structured JSON; render to one self-contained HTML file.以结构化 JSON 编写工程文档；渲染为一个自包含 HTML 文件。
  • Output输出: Inline CSS, no CDN, no external scripts. Opens in any browser, attaches to email, archives, and publishes to GitHub Pages.内联 CSS，无 CDN，无外部脚本。可在任意浏览器打开，也适合邮件附件、归档和发布到 GitHub Pages。
  • Use as使用方式: A Claude Code skill, or a standalone Python renderer with zero third-party dependencies.可作为 Claude Code skill 使用，也可作为零第三方依赖的独立 Python 渲染器使用。

CAST Docs is a controlled replacement for Markdown-based engineering documents. The JSON is the source. The HTML is the artifact. The renderer enforces one visual system, one class vocabulary, and one validation contract across every output.CAST Docs 是对 Markdown 工程文档的受控替代方案。JSON 是源文件，HTML 是产物。渲染器在所有输出中强制使用同一套视觉系统、类名词表和校验契约。

Five built-in scenarios五个内置场景

Each scenario picks a section order and a set of components that fit a common writing situation. Open any preview to see a rendered example; the matching JSON fixture lives next to it in the repository.每个场景都会选择适合常见写作情境的章节顺序和组件集合。打开任意预览即可查看渲染示例；对应的 JSON fixture 位于仓库中的相邻位置。

  • Problem investigation问题排查 - Tracking down a bug, regression, or anomaly.追踪 bug、回归或异常。
  • Option decision方案决策 - Comparing options and recording why one was chosen.比较多个方案，并记录选择某个方案的原因。
  • Cross-team alignment跨团队对齐 - Aligning multiple teams on scope and ownership.让多个团队在范围和归属上达成一致。
  • Document digest文档摘要 - Summarizing a long document or set of documents.总结长文档或一组文档。
  • Principle showcase原则展示 - Curating principles or guidelines for reuse.整理可复用的原则或指南。
  • Component gallery组件大全 - Inspect every common block, inline mark, table, diff view, and diagram in one page.在一个页面中查看所有常用 block、富文本标记、表格、diff 视图和图表。

Why this shape为什么采用这种形态

HTML, not Markdown使用 HTML，而不是 Markdown

Markdown cannot carry callouts, decision matrices, diff blocks, diagram viewers, and controlled tables without per-renderer extensions. Owning the HTML produces consistent output across every viewer.如果不依赖各渲染器扩展，Markdown 无法稳定承载 callout、决策矩阵、diff block、图表查看器和受控表格。由我们掌控 HTML，才能在所有查看器中得到一致输出。

JSON intermediateJSON 中间表示

Prose to HTML by a model is non-deterministic. Prose to JSON to HTML separates content from rendering, keeps the JSON validatable and diffable, and lets the renderer produce byte-stable output.模型直接从 prose 生成 HTML 是非确定性的。prose 到 JSON 再到 HTML 可以分离内容和渲染，让 JSON 可校验、可 diff，并让渲染器产出字节稳定的结果。

Self-contained output自包含输出

Inline CSS, no external assets. The HTML survives being opened in any browser, attached to email, copied to S3, or served from GitHub Pages.内联 CSS，无外部资源。HTML 可以在任意浏览器打开，作为邮件附件发送，复制到 S3，或通过 GitHub Pages 提供服务。

Config-driven renderer配置驱动渲染器

Adding a new component, theme, layout, or interaction is config plus an asset file, not a Python branch. Visual and structural rules live in config/. The Python in scripts/ only assembles.新增组件、主题、布局或交互时，改配置和资源文件即可，而不是增加 Python 分支。视觉和结构规则放在 config/ 中，scripts/ 里的 Python 只负责组装。

Get started开始使用

Requires Python 3.9 or newer. Standard library only. Clone the repository and render one of the bundled examples:需要 Python 3.9 或更新版本。仅使用标准库。克隆仓库并渲染一个内置示例：

SHPreset
1git clone https://github.com/CAST-docs/cast-docs.git2cd cast-docs3python3 scripts/render_html.py \4  --input examples/problem-investigation.json \5  --output out.html \6  --validate
Use as a Claude Code skill作为 Claude Code skill 使用

Clone into ~/.claude/skills/cast-docs and the skill activates when you ask Claude for an engineering spec, decision record, or self-contained HTML document. References in references/ are loaded on demand.克隆到 ~/.claude/skills/cast-docs 后，当你要求 Claude 生成工程规格、决策记录或自包含 HTML 文档时，该 skill 会自动启用。references/ 中的参考材料会按需加载。

  • READMEREADME - Installation, validation pipeline, repository layout.安装方式、校验流程和仓库结构。
  • JSON schemaJSON schema - The contract every input document must satisfy.所有输入文档都必须满足的契约。
  • Design references设计参考 - Design laws, generation contract, module architecture, interactive features.设计法则、生成契约、模块架构和交互能力。
