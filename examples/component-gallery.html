Plain Content基础内容

  • Purpose用途: Use this page to inspect the visual language and regression-test every common block.使用本页检查视觉语言，并回归测试所有常用 block。
  • Order顺序: The gallery starts with simple prose and ends with interactive diagrams.组件大全从简单 prose 开始，逐步到交互式图表。

A paragraph is the smallest prose block. It should read comfortably, preserve links, and keep spacing stable around richer components.段落是最小的 prose block。它应该易读，保留链接，并在复杂组件周围保持稳定间距。

Info callout信息提示

Callouts highlight context without replacing the surrounding explanation.Callout 用于强调上下文，但不替代周围说明。

Rich Text Marks富文本标记

Inline marks include strong, italic, code, underline, highlight, and strikethrough.行内标记包括 加粗、斜体、代码、下划线、高亮 和 删除线。

Semantic marks carry meaning: legacy field, P95 latency, bounded context, and schema reference.语义标记会携带含义：旧字段、P95 延迟、有界上下文，以及 schema 引用。

Lists And Criteria列表与验收标准

  • Unordered lists are good for scannable facts.无序列表适合可扫描的事实。
  • Items can include inline code and semantic marks.列表项可以包含 行内代码 和语义标记。
  1. First decide the document type.先决定文档类型。
  2. Then choose the scenario skeleton.再选择场景骨架。
  3. Finally render and validate.最后渲染并校验。
  1. Every visible component is generated from JSON.每个可见组件都由 JSON 生成。
  2. Both English and Chinese content switch in place.英文和中文内容可以原地切换。
