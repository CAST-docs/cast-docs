CAST Docs design principlesCAST Docs 设计原则

Summary摘要

  • Purpose目的: Keep generated HTML stable, reusable, and easy to validate.保持生成的 HTML 稳定、可复用且易于校验。

Context背景

The renderer produces deterministic documents without copying large templates into model context.渲染器会生成 确定性的 文档，无需 将大型模板复制进模型上下文。

Principles Or Features原则或特性

Consistent output一致输出

Use one shared shell and semantic classes.使用 一个共享 shell 和语义化类名。

On-demand composition按需组合

Render only selected components.只渲染被选择的组件。

Scenario skeletons场景骨架

Use configured section order.使用已配置的章节顺序。

Reuse first复用优先

Prefer existing components and schema fragments.优先使用已有组件和 schema 片段。

Examples示例

sh
1python3 scripts/render_html.py --input doc.json --output doc.html --validate
